# SpeakLow 是怎么工作的？

## 一句话总结

你按住快捷键对着麦克风说话 → 边说边识别，屏幕上实时显示 → 松开后文字自动粘贴到你的光标位置。

## 打个比方

想象你在一场会议里有个实时翻译：

1. **你**（麦克风）：用中文说话
2. **翻译**（asr-bridge）：听到你的话，实时传给后台速记员
3. **速记员**（阿里云 qwen3-asr-flash-realtime）：边听边写，一句一句把文字传回来
4. **翻译**（asr-bridge）：把写好的文字实时显示在你面前的屏幕上
5. **你的助手**（TextInserter）：等你说完，把最终版文字抄到你正在编辑的文档里
6. **润色**（LLM）：帮你修正错别字、去掉口语化的"嗯""那个"

## 完整流程图

```
你按住快捷键（右 Option 键）
  ↓
[麦克风开始录音] ← 屏幕顶部出现波形动画
  ↓ 边录边发（每 100ms 发一块音频）
[asr-bridge] ← 本地"翻译"程序，住在你的 Mac 里
  ↓ 通过 WebSocket 实时传到云端
[阿里云 qwen3-asr-flash-realtime] ← 远程的语音识别引擎
  ↓ 边听边返回识别结果
[asr-bridge → 浮窗预览] ← 你能看到识别出的文字在实时更新
  ↓
你松开快捷键
  ↓
[最终识别结果] ← 云端返回完整的最终文字
  ↓
[AI 润色] ← qwen-turbo-latest 自动纠错、去口语化（可关闭）
  ↓
[粘贴到光标位置] ← 自动插入到你正在打字的地方
```

## 为什么需要"翻译"（asr-bridge）？

阿里云的语音识别服务用的是 **WebSocket**（双向实时通信），而且需要在连接时附上 API 密钥。

macOS 的 Swift 语言在 WebSocket 连接时不能自定义 HTTP Header（技术限制），所以我们用一个 Go 写的小程序做"中间人"：

- Swift app 通过本地 WebSocket 把音频发给 asr-bridge（不需要认证）
- asr-bridge 通过带认证的 WebSocket 把音频转发给阿里云
- 阿里云识别完后告诉 asr-bridge，asr-bridge 再传回给 Swift app

## "边说边出字"是怎么实现的？

传统的语音输入是"录完再识别"，SpeakLow 用的是**流式识别**：

1. 你按住快捷键开始说话
2. 麦克风每 100ms 采集一小段音频
3. 这段音频立刻通过 WebSocket 发到阿里云
4. 阿里云边收边识别，每识别出一段文字就立刻返回
5. 你的屏幕上实时更新显示

所以你能看到文字一个一个蹦出来，而不是说完了等半天。

## 哪些在本地，哪些在云端？

| 步骤 | 在哪里 | 说明 |
|------|--------|------|
| 录音 | 本地 Mac | 用你的麦克风录制声音 |
| asr-bridge 转发 | 本地 Mac | 只是个搬运工，不消耗算力 |
| 语音识别 | 阿里云 | **核心计算在这里**，需要联网 |
| AI 文字润色 | 阿里云 | 纠错和去口语化（可关闭） |
| 文字插入 | 本地 Mac | 把识别结果粘贴到输入框 |

## 这意味着什么？

- **必须联网**才能使用（因为识别在云端）
- **需要 API 密钥**（就是 `.env` 文件里的 `DASHSCOPE_API_KEY`，相当于云端服务的通行证）
- 识别速度取决于你的**网络速度**和阿里云的响应速度
- 你的语音数据会**经过阿里云的服务器**（如果在意隐私请注意）
- 好处是**识别准确率高**，因为云端有大模型在运行
- 支持**中英文混合**识别（如"用 Cursor 写代码"）
