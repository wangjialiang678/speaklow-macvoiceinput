# SpeakLow 是怎么工作的？

## 一句话总结

你按住快捷键对着麦克风说话 → 识别文字显示在屏幕上 → 松开后文字自动粘贴到你的光标位置。

## 两种识别模式

SpeakLow 支持两种识别模式，可在**设置 → 识别**中切换，切换立即生效：

### 流式模式（默认）

边说边识别，文字实时显示在屏幕上。需要在后台运行一个名为 `asr-bridge` 的本地服务，SpeakLow 启动时会自动启动它。

适合：日常使用、追求实时反馈的场景。

### 批量模式

录完之后统一识别，识别前不会有实时预览。**不需要后台服务**，结构更简单。

适合：网络不稳定、或者只需要偶尔使用的场景。

两种模式都支持 LLM 文字润色（可在设置中关闭）。

## 流式模式：完整流程

```
你按住快捷键（右 Option 键）
  ↓
[麦克风开始录音] ← 屏幕顶部出现波形动画
  ↓ 边录边发（每 100ms 发一块音频）
[asr-bridge] ← 本地"中转"程序，住在你的 Mac 里
  ↓ 通过 WebSocket 实时传到云端
[阿里云 qwen3-asr-flash-realtime] ← 远程的语音识别引擎
  ↓ 边听边返回识别结果
[asr-bridge → 浮窗预览] ← 你能看到识别出的文字在实时更新
  ↓
你松开快捷键
  ↓
[最终识别结果] ← 云端返回完整的最终文字
  ↓
[AI 润色] ← qwen-turbo-latest 自动纠错、去口语化（可关闭）
  ↓
[粘贴到光标位置] ← 自动插入到你正在打字的地方
```

## 批量模式：完整流程

```
你按住快捷键（右 Option 键）
  ↓
[麦克风开始录音] ← 屏幕顶部出现波形动画
  ↓
你松开快捷键
  ↓
[录音文件] ← 保存为 WAV 格式
  ↓ 直接调用云端 REST API
[阿里云 qwen3-asr-flash] ← 批量语音识别引擎
  ↓
[识别结果]
  ↓
[AI 润色] ← 同流式模式（可关闭）
  ↓
[粘贴到光标位置]
```

## 为什么流式模式需要"中转"程序（asr-bridge）？

阿里云的流式语音识别服务用的是 **WebSocket**（双向实时通信），而且需要在连接时附上 API 密钥。

macOS 的 Swift 语言在 WebSocket 连接时不能自定义 HTTP Header（技术限制），所以我们用一个 Go 写的小程序做"中间人"：

- Swift app 通过本地 WebSocket 把音频发给 asr-bridge（不需要认证）
- asr-bridge 通过带认证的 WebSocket 把音频转发给阿里云
- 阿里云识别完后告诉 asr-bridge，asr-bridge 再传回给 Swift app

批量模式直接调用 REST API，Swift 原生支持，不需要这个中转程序。

## "边说边出字"是怎么实现的？（流式模式）

传统的语音输入是"录完再识别"，SpeakLow 流式模式用的是**实时流式识别**：

1. 你按住快捷键开始说话
2. 麦克风每 100ms 采集一小段音频
3. 这段音频立刻通过 WebSocket 发到阿里云
4. 阿里云边收边识别，每识别出一段文字就立刻返回
5. 你的屏幕上实时更新显示

所以你能看到文字一个一个蹦出来，而不是说完了等半天。

## 哪些在本地，哪些在云端？

| 步骤 | 在哪里 | 说明 |
|------|--------|------|
| 录音 | 本地 Mac | 用你的麦克风录制声音 |
| asr-bridge 转发（仅流式模式） | 本地 Mac | 只是个搬运工，不消耗算力 |
| 语音识别 | 阿里云 | **核心计算在这里**，需要联网 |
| AI 文字润色 | 阿里云 | 纠错和去口语化（可关闭） |
| 文字插入 | 本地 Mac | 把识别结果粘贴到输入框 |

## 这意味着什么？

- **必须联网**才能使用（因为识别在云端）
- **需要 API 密钥**（在设置 → 密钥 中配置，相当于云端服务的通行证）
- 识别速度取决于你的**网络速度**和阿里云的响应速度
- 你的语音数据会**经过阿里云的服务器**（如果在意隐私请注意）
- 好处是**识别准确率高**，因为云端有大模型在运行
- 支持**中英文混合**识别（如"用 Cursor 写代码"）
