# SpeakLow 是怎么工作的？

## 一句话总结

你对着麦克风说话 → 录音发到阿里云 → 阿里云把语音变成文字 → 文字自动粘贴到你的光标位置。

## 打个比方

想象你去一家餐厅点菜：

1. **你**（麦克风）：用中文说出你想吃什么
2. **服务员**（asr-bridge）：听到你的话，走到后厨转告给厨师
3. **厨师**（阿里云 FunASR）：把你说的话写成菜单（文字）
4. **服务员**（asr-bridge）：把写好的菜单拿回来给你
5. **你的手**（TextInserter）：把菜单上的字抄到你正在编辑的文档里

## 为什么需要"服务员"（asr-bridge）？

阿里云的语音识别服务有个特殊要求：它只接受一种叫 **WebSocket** 的通信方式。

你可以把通信方式想象成打电话和发短信的区别：

- **普通 HTTP 请求**（发短信）：发一条，收一条，结束。简单直接。
- **WebSocket**（打电话）：双方一直保持连线，可以你一句我一句地聊。

阿里云选择了"打电话"的方式，因为语音识别需要一边收音频一边返回结果。

**问题是**：苹果的 Swift 语言在"打电话"时有个限制 —— 不能在接通的时候报上你的身份（API 密钥）。就像打电话时没法在拨号的同时递上名片。

所以我们需要一个"服务员"（asr-bridge）作为中间人：
- Swift app 用"发短信"的简单方式把录音交给服务员
- 服务员用"打电话"的方式把录音转交给阿里云，并且帮你递上名片
- 阿里云识别完后告诉服务员，服务员再把结果"发短信"回给 Swift app

## 完整流程图

```
你说话
  ↓
[麦克风录音] ← 在你的 Mac 上完成
  ↓
[转换格式] ← 把录音转成阿里云能听懂的格式（16kHz）
  ↓
[asr-bridge] ← 本地的"服务员"程序，住在你的 Mac 里
  ↓ （通过网络发送到云端）
[阿里云 FunASR] ← 远程的语音识别引擎，算力在云端
  ↓ （识别结果通过网络返回）
[asr-bridge] ← "服务员"收到文字结果
  ↓
[粘贴到光标位置] ← 自动 Cmd+V 粘贴到你正在打字的地方
```

## 哪些在本地，哪些在云端？

| 步骤 | 在哪里 | 说明 |
|------|--------|------|
| 录音 | 本地 Mac | 用你的麦克风录制声音 |
| 格式转换 | 本地 Mac | 把录音转成标准格式 |
| asr-bridge 转发 | 本地 Mac | 只是个搬运工，不消耗算力 |
| 语音识别 | 阿里云 | **核心计算在这里**，需要联网 |
| 文字插入 | 本地 Mac | 把识别结果粘贴到输入框 |

## 这意味着什么？

- **必须联网**才能使用（因为识别在云端）
- **需要 API 密钥**（就是 `.env` 文件里的 `DASHSCOPE_API_KEY`，相当于云端服务的通行证）
- 识别速度取决于你的**网络速度**和阿里云的响应速度
- 你的语音数据会**经过阿里云的服务器**（如果在意隐私请注意）
- 好处是**识别准确率高**，因为云端有大模型在运行
